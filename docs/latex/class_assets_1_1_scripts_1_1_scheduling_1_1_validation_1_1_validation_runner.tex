Verifies correctness invariants of the DESSimulator against a benchmark instance. 

Runs a fresh simulation with {\ttfamily rule} and performs four sequential checks. All violations found across all checks are collected and returned — execution does not stop at the first error. A correct implementation should always return an empty list.

The four checks are:\+
\begin{DoxyEnumerate}
\item {\bfseries{Valid times}}:\+ every Operation has non-\/negative Operation.\+Start\+Time and Operation.\+End\+Time. A value of {\ttfamily -\/1} indicates the operation was never scheduled.
\item {\bfseries{Precedence}}:\+ for each job, operation {\ttfamily i} must start no earlier than operation {\ttfamily i-\/1} ends. Violations indicate a sequencing bug in DESSimulator.
\item {\bfseries{No machine overlap}}:\+ no two operations assigned to the same machine may overlap in time. Operations are sorted by start time and checked pairwise. Violations indicate a machine contention bug.
\item {\bfseries{Makespan lower bound}}:\+ the achieved makespan must be at or above Taillard\+Metadata.\+lower\+\_\+bound. A violation indicates a bug that produces a schedule that is theoretically impossible.
\end{DoxyEnumerate}

\begin{DoxyNote}{Note}
A floating-\/point epsilon of {\ttfamily 1e-\/9} is applied to all time comparisons to guard against rounding errors in double arithmetic.
\end{DoxyNote}
