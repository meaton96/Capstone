Detailed Description

Represents a single machine on the shop floor.

Maintains the machine's current MachineState, the Operation it is actively processing, and a WaitingQueue of operations that have arrived but cannot yet start. State transitions are driven exclusively by DESSimulator via StartProcessing and FinishProcessing.

◆ FinishProcessing()

void Assets.Scripts.Scheduling.Core.Machine.FinishProcessing ( )

Transitions the machine to MachineState.Idle after an operation completes.

Clears CurrentOperation and sets State to MachineState.Idle. Does not select the next operation from WaitingQueue — that responsibility belongs to DESSimulator.TryStartNextOnMachine, which is called immediately after this method in DESSimulator.HandleOperationComplete.

◆ StartProcessing()

void Assets.Scripts.Scheduling.Core.Machine.StartProcessing ( Operation op,
double currentTime )

Transitions the machine to MachineState.Busy and begins processing an operation.

Sets State to MachineState.Busy, assigns op to CurrentOperation, stamps Operation.StartTime with currentTime, and computes Operation.EndTime as currentTime + Operation.Duration. BusyUntil is updated to match Operation.EndTime so the simulator can schedule the EventType.OperationComplete event.

Parameters
opThe operation to begin processing. Must not be null.
currentTimeThe current simulation time at which processing starts.

Ordered list of operations waiting to be processed on this machine.
