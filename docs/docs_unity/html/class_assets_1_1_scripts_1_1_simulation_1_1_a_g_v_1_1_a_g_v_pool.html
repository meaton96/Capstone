◆ InitializeFleet()

void Assets.Scripts.Simulation.AGV.AGVPool.InitializeFleet ( )

Spawns and prepares the AGV fleet at the designated parking area.

Clears any existing fleet instances and the pending request queue. Instances are staggered along the X-axis starting from the layout manager's parking position. Each AGV is initialized with a unique ID and a callback for idle state transitions.

Postcondition
The fleet list is populated with initialized AGVControllers and pendingRequests is empty.

Attempts to assign a transport task to an available AGV or queues it if the fleet is full.

Queries the fleet for an idle AGV. If one is found, the job is dispatched immediately. If all AGVs are busy, the task is stored in pendingRequests to be processed once an AGV becomes idle.

Parameters
jobIdThe unique ID of the job.
pickupPosThe world position for pickup.
dropoffSlotPosThe pre-reserved world position for dropoff.
sourceThe source machine.
dropoffThe destination machine.
Postcondition
Job is either assigned to an AGV or added to the tail of the queue.

Initiates a job dispatch with a calculated delay.

Offloads the dispatch to a coroutine that waits for a duration based on the dispatchIndex. This prevents CPU spikes caused by multiple AGVs calculating complex BFS paths on the same frame during simulation startup.

Parameters
jobIdThe unique ID of the job.
pickupPosThe world position for pickup.
dropoffSlotPosThe pre-reserved world position for dropoff.
sourceThe source machine.
dropoffThe destination machine.
dispatchIndexThe index used to calculate the stagger delay (index * 0.3s).
