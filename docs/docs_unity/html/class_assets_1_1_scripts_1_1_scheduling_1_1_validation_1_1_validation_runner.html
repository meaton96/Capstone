Detailed Description

Validates the DESSimulator against Taillard JSP benchmark instances.

Provides three levels of validation:

  • RunSingle — simulate one instance with one DispatchingRule and measure the gap to optimum.
  • RunAllRules — sweep every DispatchingRule and return results sorted by makespan.
  • ValidateConstraints — verify correctness invariants (valid times, precedence, no machine overlap, makespan above lower bound).

Usage outside Unity:

var runner = new ValidationRunner();
runner.RunValidation(taillardInstance);

List< ValidationResult > Assets.Scripts.Scheduling.Validation.ValidationRunner.RunAllRules ( TaillardInstance instance)

Runs every DispatchingRule on an instance and returns results sorted by makespan.

Iterates all values of the DispatchingRule enum via Enum.GetValues, delegates each to RunSingle, and sorts the collected results by ValidationResult.Makespan ascending so the best-performing heuristic appears first. Useful for identifying the strongest baseline rule for a given instance before applying RL-based scheduling.

Parameters
instanceThe benchmark instance to run all rules against.
Returns
A list of ValidationResult values, one per DispatchingRule, sorted by makespan ascending.

DispatchingRule rule )

Simulates a single instance with one dispatching rule and returns a gap analysis.

Constructs a fresh DESSimulator, loads instance, sets rule as the active dispatching rule, runs to completion, and packages the result into a ValidationResult. The gap percentage is computed relative to TaillardMetadata.optimum and will be infinite or undefined if optimum is 0.

Parameters
instanceThe benchmark instance to simulate. Must have valid TaillardInstance.metadata with a non-zero TaillardMetadata.optimum for gap analysis to be meaningful.
ruleThe DispatchingRule to apply during the simulation run.
Returns
A ValidationResult containing the makespan, gap, and bound values for this rule and instance combination.

List< string > Assets.Scripts.Scheduling.Validation.ValidationRunner.ValidateConstraints ( TaillardInstance instance,
DispatchingRule rule )

Verifies correctness invariants of the DESSimulator against a benchmark instance.

Runs a fresh simulation with rule and performs four sequential checks. All violations found across all checks are collected and returned — execution does not stop at the first error. A correct implementation should always return an empty list.

The four checks are:

  1. Valid times: every Operation has non-negative Operation.StartTime and Operation.EndTime. A value of -1 indicates the operation was never scheduled.
  2. Precedence: for each job, operation i must start no earlier than operation i-1 ends. Violations indicate a sequencing bug in DESSimulator.
  3. No machine overlap: no two operations assigned to the same machine may overlap in time. Operations are sorted by start time and checked pairwise. Violations indicate a machine contention bug.
  4. Makespan lower bound: the achieved makespan must be at or above TaillardMetadata.lower_bound. A violation indicates a bug that produces a schedule that is theoretically impossible.
Note
A floating-point epsilon of 1e-9 is applied to all time comparisons to guard against rounding errors in double arithmetic.
Parameters
instanceThe benchmark instance to validate against. Must have valid TaillardMetadata with a meaningful TaillardMetadata.lower_bound.
ruleThe DispatchingRule to use for the validation simulation run.
Returns
A list of human-readable error strings describing each violation found. Returns an empty list if all constraints are satisfied.
