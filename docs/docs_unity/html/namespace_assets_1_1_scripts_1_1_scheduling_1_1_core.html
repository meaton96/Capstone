◆ DispatchingRule

enum Assets.Scripts.Scheduling.Core.DispatchingRule

Composite primary dispatching rules for machine queue selection.

Tiebreaking is always by lowest job ID to match job_shop_lib's default internal ordering, ensuring identical makespans when the primary sort key is equal. Each enumerator name encodes its primary heuristic and secondary metric separated by an underscore, except MOR which has no secondary metric.

Enumerator
SPT_SMPT 

Shortest Processing Time — primary sort by Operation.Duration ascending.

SPT_SRWT 

Shortest Processing Time with Shortest Remaining Work Time secondary metric.

LPT_MMUR 

Longest Processing Time — primary sort by Operation.Duration descending.

LPT_SMPT 

Longest Processing Time with Smallest Most Urgent Remaining Time secondary metric.

SRT_SRWT 

Shortest Remaining Time — primary sort by total remaining work ascending.

SRT_SMPT 

Shortest Remaining Time with Smallest Most Urgent Remaining Time secondary metric.

LRT_MMUR 

Longest Remaining Time — primary sort by total remaining work descending. Equivalent to most_work_remaining in job_shop_lib.

SDT_SRWT 

Smallest Due Time — FIFO based on arrival order. Equivalent to first_come_first_served in job_shop_lib.

MOR 

Most Operations Remaining — primary sort by count of unprocessed operations descending. Equivalent to most_operations_remaining in job_shop_lib.

◆ MachineState

enum Assets.Scripts.Scheduling.Core.MachineState

Operational states a machine can occupy during simulation.

MachineState.Failed and MachineState.Repair are reserved for future fault-injection extensions and are not currently handled by DESSimulator.

Enumerator
Idle 

Machine is free and ready to accept a new operation.

Busy 

Machine is currently processing an operation.

Failed 

Machine has experienced a fault and cannot accept work. Reserved for future use.

Repair 

Machine is undergoing repair after a fault. Reserved for future use.

◆ SimStepResult

enum Assets.Scripts.Scheduling.Core.SimStepResult

Result of a single event-processing step.

Returned by DESSimulator.ProcessNextEvent to tell the caller whether the simulation needs an external dispatch decision, can continue autonomously, or has finished.

Enumerator
Continue 

Event was processed and more events remain. No decision needed.

DecisionRequired 

A machine has become free with a non-empty queue. The caller must supply a DispatchingRule via DESSimulator.ApplyDecision before calling ProcessNextEvent again.

Done 

The event queue is empty. All jobs are complete.
