Detailed Description

Unity entry point for loading and validating Taillard JSP benchmark instances.

Attach to any GameObject in the scene. On Start, or manually via the "Run Validation" context menu entry, the component loads a TaillardInstance from Resources/Instances/, runs constraint checks through ValidationRunner, executes one or all DispatchingRule variants, and optionally prints a Gantt schedule to the console via DESSimulator.PrintSchedule.

Note
JSON files must be placed at Assets/Resources/Instances/ <name>.json and referenced by filename without the .json extension via instanceFileName.

void Assets.Scripts.Scheduling.Unity.TaillardValidator.LoadAndValidate ( )

Loads the configured Taillard instance and runs the full validation pipeline.

Executes four sequential steps:

  1. Load the JSON instance from Resources/Instances/ via LoadInstance. Aborts early if the file is missing or cannot be parsed.
  2. Validate scheduling constraints (precedence, no machine overlap, bound checks) using ValidationRunner.ValidateConstraints with DispatchingRule.SPT_SMPT. All violations are logged as errors via Debug.LogError.
  3. Run dispatching rules: all rules via ValidationRunner.RunAllRules if runAllRules is true, otherwise a single rule via ValidationRunner.RunSingle. Results are colour-coded green / yellow / red based on gap percentage.
  4. Optionally print a Gantt schedule to the console if printSchedule is true, using a fresh DESSimulator run with singleRule.
Note
Also callable from the Unity Inspector via the "Run Validation" context menu without entering Play Mode.
