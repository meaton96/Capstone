Detailed Description

Generates ground-truth makespans and operation schedules from job_shop_lib for cross-validation against the C# DES simulator.

Runs every rule in RULES against every instance in INSTANCES using job_shop_lib's DispatchingRuleSolver, then writes two output files whose structure is kept deliberately in sync with the C# ValidationExporter outputs:

  • reference_makespans.csv — one row per instance/rule combination, with columns matching csharp_makespans.csv for direct DataFrame merging in compare_results.py.
  • reference_schedules.json — full operation-level schedule per instance/rule, keyed as "{instance}_{rule_key}", for divergence analysis in compare_results.py and compare_mirror.py.
Prerequisites
pip install job-shop-lib pandas
Usage
python generate_reference.py
Output
  • reference_makespans.csv — ground-truth makespans for all instance/rule pairs.
  • reference_schedules.json — per-instance/rule operation lists sorted by start time, then machine ID, then job ID.
  • A pivot table printed to stdout summarising makespan by instance × rule.
Note
The C# simulator should produce identical makespans for all mapped rules. If not, the schedule JSON lets compare_results.py identify exactly which operation first diverges between the two implementations.

Variable Documentation

◆ instance_schedule_key

str generate_reference.instance_schedule_key = f"{instance_name}_{rule_key}"

Extract the full operation-level schedule for this instance/rule pair.

schedule.schedule is a list of lists indexed by machine ID. Each inner list contains ScheduledOperation objects whose .operation field exposes position_in_job and duration. The extracted dicts are sorted by start time, then machine ID, then job ID — matching the sort order used by ValidationExporter.RunAndExport — so that compare_results.py can diff the two schedule JSONs entry-by-entry without re-sorting.

◆ INSTANCES

list generate_reference.INSTANCES = ["ta01", "ta11", "ta21", "ta31"]

Taillard benchmark instance names to process.

Kept small by default for fast iteration during debugging. Expand to the full Taillard set (e.g. ta01 – ta80) once the C# and Python outputs are confirmed to match on this subset.

◆ metadata

generate_reference.metadata = instance.metadata if hasattr(instance, "metadata") else {}

Extract the optimum from instance metadata if available.

job_shop_lib exposes metadata as a dict on some instance types. The hasattr guard handles instances that do not carry metadata, in which case optimum is None and gap percentage is skipped for that instance.

◆ records

Accumulates one record dict per instance/rule combination for CSV output.

Populated during the main simulation loop and converted to a DataFrame at the end. Each entry matches the column schema of csharp_makespans.csv.

Maps job_shop_lib full rule names to the short keys used in both CSV outputs.

Short keys must match the key field in ValidationExporter.RuleMap exactly so that compare_results.py can merge the two DataFrames on the rule column without any name translation. Adding a rule here without a corresponding entry in ValidationExporter.RuleMap will produce unmatched rows in the comparison.

Fully qualified job_shop_lib rule names to simulate.

Only rules that have a confirmed 1:1 mapping to a C# DispatchingRule are included. The inline comments document the corresponding C# enum value for each rule. Rules without a C# equivalent (e.g. most_operations_remaining) can be included here for reference but will produce unmatched rows in the comparison.

◆ schedules

Accumulates full operation schedules keyed by "{instance}_{rule_key}".

Populated during the main simulation loop and serialized to reference_schedules.json. Key format must match ValidationExporter.RunAndExport and mirror_des.py so that compare_results.py can look up schedules by the same key used in the merged DataFrame.
