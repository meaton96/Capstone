Spawns and prepares the \doxylink{namespace_assets_1_1_scripts_1_1_simulation_1_1_a_g_v}{AGV} fleet at the designated parking area. 

Clears any existing fleet instances and the pending request queue. Instances are staggered along the X-\/axis starting from the layout manager\textquotesingle{}s parking position. Each \doxylink{namespace_assets_1_1_scripts_1_1_simulation_1_1_a_g_v}{AGV} is initialized with a unique ID and a callback for idle state transitions.

\begin{DoxyPostcond}{Postcondition}
The fleet list is populated with initialized AGVControllers and pending\+Requests is empty. 
\end{DoxyPostcond}

Attempts to assign a transport task to an available \doxylink{namespace_assets_1_1_scripts_1_1_simulation_1_1_a_g_v}{AGV} or queues it if the fleet is full. 

Queries the fleet for an idle \doxylink{namespace_assets_1_1_scripts_1_1_simulation_1_1_a_g_v}{AGV}. If one is found, the job is dispatched immediately. If all AGVs are busy, the task is stored in pending\+Requests to be processed once an \doxylink{namespace_assets_1_1_scripts_1_1_simulation_1_1_a_g_v}{AGV} becomes idle.

Initiates a job dispatch with a calculated delay. 

Offloads the dispatch to a coroutine that waits for a duration based on the {\ttfamily dispatch\+Index}. This prevents CPU spikes caused by multiple AGVs calculating complex BFS paths on the same frame during simulation startup.
