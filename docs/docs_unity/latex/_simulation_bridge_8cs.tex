\doxysubsection{Detailed Description}
Stepped DES orchestrator that pauses at every dispatch decision point, accepts an action (from a fixed rule or RL agent), then advances to the next. 

The simulation loop is:\+

This is the Gymnasium {\ttfamily step()} pattern. The bridge owns the DES, the visual layer, and the reward computation. It exposes the same interface whether the caller is a fixed-\/rule validator (Phase 1), an ML-\/\+Agents policy (Phase 3), or a Python training loop over a socket (Phase 3 alt).

\begin{DoxyParagraph}{Execution modes}

\begin{DoxyItemize}
\item {\bfseries{Training mode}} (default):\+ No visuals. Step returns synchronously. Maximum throughput for PPO data collection.
\item {\bfseries{Visual debug mode}}:\+ Visuals are driven between decision points via a coroutine. Step still returns immediately, but visual playback catches up asynchronously. Toggle with enable\+Visuals. 
\end{DoxyItemize}
