Detailed Description

Immutable record representing a single scheduled event in the simulation timeline.

Implements IComparable<SimEvent> so that instances can be stored directly in a SortedSet inside EventQueue. Ordering is by Time ascending, with SequenceNumber as a FIFO tiebreaker for events scheduled at the same instant.

int Assets.Scripts.Scheduling.Core.SimEvent.CompareTo ( SimEvent other)

Compares this event to another for priority queue ordering.

Primary sort is by Time ascending. When two events share the same time, SequenceNumber is used as a FIFO tiebreaker, ensuring earlier-enqueued events are processed first.

Parameters
otherThe event to compare against.
Returns
A negative value if this event should be processed before other, zero if they are equal, or a positive value if other has higher priority.

Monotonically increasing counter assigned at enqueue time.

Used as a FIFO tiebreaker so that events with identical Time values are processed in the order they were enqueued. A lower sequence number indicates higher priority.
